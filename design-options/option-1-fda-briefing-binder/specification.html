Executive view for signoff

This page is written as a reviewable record for scientific and regulatory stakeholders. It emphasizes current shipped behavior, explicit limitations, and where signoff should remain conditional.

Actual shipped operator surface

The current operator surface is presented as a bounded, source-grounded description of commands, artifact paths, and output contracts rather than roadmap intent.

Scientific workflow actually implemented

The workflow section describes the implemented sequence of selection, simulation, scoring, calibration, or reporting as evidenced by current source code and tests.

What scientific reviewers should separate

Reviewers should separate shipped code, generated artifacts, test-only or synthetic proof paths, and future or partially materialized claims.

Current scientific and operational limits

Known limitations, host/tool dependencies, and scientific authority boundaries are elevated rather than hidden.

Recommended signoff questions

The page closes with concrete reviewer prompts that can be used for governance, validation, and release readiness discussions.

Evidence appendix

All substantive claims should resolve to exact files, commands, tests, or artifact families so the page remains auditable.
